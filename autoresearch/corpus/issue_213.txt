Wire Thompson sampling into live PairsEngine

## Problem

`engine/crates/pair-picker/src/thompson.rs` has a complete Thompson sampling implementation for pair selection, but it is **not integrated** into the live PairsEngine. The code exists, compiles, and is tested — but never called from the trading path.

## What Thompson sampling does

- Maintains Beta(α, β) posteriors per pair based on trade outcomes (win/loss)
- Samples from posteriors to rank pairs for capital allocation
- Naturally balances exploration (untested pairs) vs exploitation (proven pairs)

## Fix

1. On trade close in PairsEngine, update Thompson posterior for that pair (win/loss based on P&L > 0)
2. When multiple pairs signal simultaneously, use Thompson sampling to prioritize which get capital
3. Persist posteriors to disk (alongside `active_pairs.json`) so they survive restarts

## Files
- `engine/crates/pair-picker/src/thompson.rs` — existing implementation
- `engine/crates/core/src/pairs/engine.rs` — wire in at trade close + signal prioritization
- `trading/thompson_state.json` — persistence file

## Acceptance criteria
- [ ] Trade outcomes update Thompson posteriors
- [ ] Simultaneous signals ranked by Thompson sample
- [ ] State persists across restarts
- [ ] Test: pair with 80% win rate gets prioritized over 50% pair after sufficient history
