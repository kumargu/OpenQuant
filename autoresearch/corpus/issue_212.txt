P5: Kalman filter for dynamic hedge ratio estimation

## Parent
Tracked in #206 (P5)

## Problem

Static hedge ratios (even TLS from P1) drift as the cointegration relationship evolves. The current approach reloads from `active_pairs.json` periodically, but between reloads the ratio is stale. For pairs with unstable betas, this means the spread is miscalculated and z-scores drift.

## Fix

Implement a Kalman filter that updates the hedge ratio online with each new bar:
- State: `[alpha, beta]` (intercept + hedge ratio)
- Observation: `ln(price_A)` explained by `ln(price_B)`
- Process noise (Q) controls adaptation speed — tune via half-life
- Measurement noise (R) from spread residual variance

## Evidence

- **Both researchers mention this as the highest-Sharpe improvement**
- **Pole (2007)**: "Statistical Arbitrage" — Kalman filter hedge ratios, Sharpe ~3.0
- **Elliott et al. (2005)**: mean-reverting spread model with Kalman filter state estimation
- **Montana et al. (2009)**: flexible least squares for time-varying cointegration
- **ArbitrageLab**: implements Kalman hedge ratio as recommended approach

## Complexity

Large — this touches the online estimation path and changes how spreads are computed on every bar. Needs careful testing against static ratios to prove improvement.
